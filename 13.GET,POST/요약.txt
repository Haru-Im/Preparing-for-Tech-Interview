GET:
개념: GET은 웹 리소스를 가져오기 위한 HTTP 메서드입니다. 주로 웹 페이지를 불러오거나, 특정 데이터를 조회할 때 사용됩니다.

데이터 흐름: GET 요청 시 데이터(파라미터)는 URL의 일부로 전송됩니다. 
예를 들어, https://example.com/search?query=test에서 query=test 부분이 GET 요청의 데이터입니다.
제한사항: URL의 길이 제한 때문에 많은 양의 데이터를 전송하기에는 부적합합니다. 
또한, 데이터가 URL에 노출되기 때문에 민감한 정보(예: 비밀번호) 전송에는 적합하지 않습니다.


POST:
개념: POST는 리소스를 생성하거나 수정하기 위한 HTTP 메서드입니다. 
예를 들어, 회원 가입이나 폼 데이터 제출 등에 주로 사용됩니다.

데이터 흐름: POST 요청 시 데이터는 HTTP 요청의 본문(body)에 포함되어 전송됩니다. 
따라서 URL에 데이터가 노출되지 않으며, GET 방식에 비해 많은 양의 데이터 전송이 가능합니다.

보안성: 데이터가 URL에 노출되지 않기 때문에 GET보다는 보안성이 높습니다. 
그러나 HTTPS와 같은 암호화 방식을 사용하지 않으면 본문의 내용도 중간에서 쉽게 열람될 수 있으므로, 민감한 정보 전송 시에는 HTTPS를 함께 사용하는 것이 좋습니다.

GET과 POST 외에도 PUT, DELETE, PATCH 등 다양한 HTTP 메서드가 있지만, 주로 웹폼 데이터의 전송과 관련하여 GET과 POST가 가장 많이 사용됩니다.
