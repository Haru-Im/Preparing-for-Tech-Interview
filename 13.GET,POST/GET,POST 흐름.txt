Q. GET, POST의 개념과 함께 데이터 흐름에 대해서 설명해주세요.

GET:

데이터 흐름:
1. 사용자(클라이언트)가 웹 브라우저에 정보를 요청하려 할 때, 해당 정보의 파라미터를 URL 뒤에 붙여서 전송합니다.
2. 이 때 파라미터는 키=값 형태의 쌍을 이루며, 여러 개의 파라미터는 &로 구분됩니다.
3. 예를 들어, 사용자가 "apple"과 "banana"라는 키워드로 검색을 요청하면, 
요청 URL은 https://example.com/search?keyword1=apple&keyword2=banana와 같이 구성됩니다.
4. 웹 서버는 이 URL을 받아 파라미터를 분석하고, 해당하는 응답을 사용자에게 전송합니다.


POST:

데이터 흐름:
1. 사용자가 웹 페이지의 폼에 정보를 입력하고 제출 버튼을 클릭하면, 해당 정보는 HTTP 요청의 본문(body) 내부에 포함되어 웹 서버로 전송됩니다.
2. 이 데이터는 일반적으로 키=값 형태의 쌍을 이루며, 헤더에 Content-Type으로 그 형식이 명시됩니다. 
    (예: application/x-www-form-urlencoded, multipart/form-data 등)
3. 예를 들어, 사용자가 이름과 이메일을 입력하여 폼을 제출하면, 해당 데이터는 본문 내에 name=JohnDoe&email=john@example.com 형태로 전송됩니다.
4. 웹 서버는 요청의 본문을 읽어 데이터를 파싱하고, 필요한 처리를 한 후 적절한 응답을 사용자에게 전송합니다.
