Q. GET, POST 방식의 차이점에 대해서 설명해주세요.

A.

GET, POST는 서버와 클라이언트 간의 데이터를 교환할 때 사용되는 기본적인 HTTP메서드입니다.
GET과 POST는 통신 목적, 데이터 흐름 등에서 차이점이 있습니다.

우선, GET은 주로 데이터 조회에 사용되는 메서드입니다.
- 데이터 전송 방식 (데이터 흐름)
    - GET 요청 시 데이터는 파라미터 형태, 즉 URL의 일부로 전송됩니다.
    - 예를 들어, https://example.com/search?query=test에서 query=test 부분이 GET 요청의 데이터입니다.

- 특징
    - URL에 데이터가 그대로 노출
    - URL에 길이 제한이 있어 많은 양의 데이터를 보내기에는 적합하지 않음
    - 민감한 정보를 전송하기에는 적합하지 않음 (EX. 비밀번호, 개인정보)


반면, POST는 주로 데이터 생성 및 수정에 사용되는 메서드입니다.
- 데이터 전송 방식
    - POST 요청 시 데이터는 HTTP 요청의 본문(body)에 포함되어 전송됩니다.
        -> 데이터의 길이나 형식에 대한 제한이 거의 없습니다.
        -> URL에 데이터가 노출되지 않습니다. -> 보안성이 높아짐
            -> 하지만 HTTPS와 같은 암호화 방식을 사용하지 않으면 body의 내용도 쉽게 열람될 수 있음
            -> 따라서, 민감한 정보를 전송할 때는 HTTPS를 함께 사용하는 것이 좋음

- 특징
    - 데이터가 URL에 노출되지 않아 GET보다는 보안성이 좋으나 HTTPS와 같은 암호화 방식을 사용하지 않으면 이 또한 유출 가능성이 있음
    - 민감한 정보나 큰 데이터를 전송하는 데 적합함
    - HTTP 헤더의 'Content-type' 필드를 통해 데이터 타입 (ex. application/json, multipart/form-data) 등을 명시할 수 있음
