Q. http, https의 차이점에 대해 설명하라

A.
[주요 차이점]
HTTP와 HTTPS는 웹 브라우저와 웹 서버 간의 정보를 교환하는 데 사용되는 프로토콜.
이 두 프로토콜은 '보안' 측면에서 차이를 가짐 

1. HTTP (HyperText Transfer Protocol)
   - 기본적인 웹 통신 프로토콜, 웹 브라우저와 웹 서버 간의 정보 교환을 위해 사용
   - HTTP는 암호화되지 않은 텍스트 형태로 데이터를 전송. 따라서 중간에 데이터를 가로채면, 내용을 그대로 읽을 수 있음

2. HTTPS (HyperText Transfer Protocol Secure)
   - HTTP와 기본적으로 동일한 프로토콜이지만, 추가로 SSL/TLS라는 보안 기술을 사용하여 데이터를 암호화
   - 이 덕분에, 중간에 데이터를 가로채도 내용을 파악하기 어려움


[SSL, TLS란?]
   - SSL(보안 소켓 계층)과 TLS(전송 계층 보안)는 웹 통신을 보안하기 위한 방법 (암호화하는 방법)
   - 이 방법들을 이용하면 데이터를 다른 사람이 읽지 못하게 암호화할 수 있음
   - HTTP에 이 보안 방법을 추가하면, 그것이 HTTPS가 되는 것

[프로토콜 스택 중 어디에 속하는지?]
- HTTP와 HTTPS는 OSI 7계층 중 응용 계층에 속함
- 응용 계층 (Application Layer)
    : 최종 사용자와 직접 관련된 서비스와 애플리케이션의 대부분을 포함

[주로 사용하는 것]
- HTTPS
- 보안 측면에서 HTTPS가 데이터를 암호화하므로 더 안전함
- 구글 같은 검색 엔진이 HTTPS를 사용하는 사이트에 더 좋은 순위를 부여하기도 함
